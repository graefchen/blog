Writing small programs

08 March 2024

I like writing small programs, that means, programs that do one thing and this thing well, while not having a large code base and that aren't build on many dependencies.

Introduction

Before we start, we must clarify how to define small programs, or at least how I define them. For me small programs hasn't a large codebase. Optionally it is just one file big with preferably under 100 lines.

At maximum the codebase should have 1000 lines of code. The exception are dependencies. Additionally the program could have been written in a small amount of time, but this is not strictly necessary in my opinion.

A hack to get away with writing small programs is creating an (one-file) library that you add as an dependency that helps you with making programs smaller, especially when writing duplicate code.

An example

To better clarify what I mean I share the following 13 line code: 

#SingleInstance Force

; AltGr + y
<^>!y::
{
  SendText "»"
}

; AltGr + x
<^>!x::
{
  SendText "«"
}

Here I am using AutoHotkey to create a small program that enables me to use Guillemet on my German keyboard.

Writing small programs can be challenging and fun and a you should try to do it at least once.
